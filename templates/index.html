This is a web competition-simulator, where user-programmed AI bots compete for the title of: 'The World's Best Rock-Paper-Scissors AI Bot'!

How it works:

User-created bot's will play 10,000 games each against every other bot currently entered into the competition. The win percent of each bot will be displayed in the result tables, along with the opponent bot that it fared the best against, as well as the worst.

The bot with the highest overall win percentage will be declared the champion and earn the title of: 'The World's Best Rock-Paper-Scissors AI Bot' (results may vary between each execution of the program, so if you win, enjoy it while it lasts).

AI Bot Regulations:

You may create your very own AI bot for the competition by writing a JavaScript function that follows the guidelines below:

  • The function must be named after your team name.
  • The function/team must not have the same name as another entry (check the table of contenders to make sure).
  • The function/team name must not be offensive in any regard.
  • The function/team name must be of a reasonable character length (no specific character limit to allow for name creativity, but just refrain from unnecessary characters).
  • The function/team name must start with an alphabetical character and must not contain any illegal characters (improperly named functions/teams will be submitted with the resulting name after illegal characters are removed, assuming it then still abides by other naming rules).
  • The function must accept only 1 paramater (which is the previous result which the oponent returned, see return structure below).
  • The function must be prepared for the input parameter to be null during the first call/game.
  • The function must not access any external elements (by using DOM selectors or any other method).
  • The function must be commented where necessary (no un-explained black magic).
  • If the function uses local/session storage it must store all data under one key, which must be the function's name (you may store separate data in the form of nested lists/arrays or any other means that doesn't violate this rule).
  • The function must return it's action choice in the form of a string, for consistency it must be an identical case-sensitive match to one of the following options: 'rock', 'paper', 'scissors'.
  • The function must not attempt any malicious behaviour, any submissions believed to make such an attempt will be rejected and the user permanently blacklisted from re-submitting.
